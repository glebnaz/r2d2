# Планирование напоминаний

## Как работает scheduler

Scheduler запускается при старте бота и работает в бесконечном цикле с тремя триггерами:

1. **Периодический ресканоирование** — каждые N минут (по умолчанию 5) сканирует vault заново
2. **Полночь** — сбрасывает все отправленные напоминания и строит расписание на новый день
3. **Контекст отменён** — graceful shutdown

## Типы напоминаний

### Утренний дайджест
- Задачи с `due: YYYY-MM-DD` (без времени) + просроченные
- Отправляется в `morning_hour` (по умолчанию 9:00)
- Если бот стартовал после 9:00 — отправляется сразу
- Формируется через `digest.Engine` из всех зарегистрированных коллекторов

### Напоминание по времени
- Задачи с `due: YYYY-MM-DDTHH:mm`
- Отправляется ровно в указанное время через `time.AfterFunc`
- Если время уже прошло — пропускается

## Дедупликация

Каждое напоминание имеет ключ:
- Дайджест: `digest:2026-03-31`
- Timed: `/path/to/file.md:2026-03-31T16:00`

Ключи хранятся в памяти. Сбрасываются в полночь.

При ошибке отправки ключ удаляется — повторная попытка при следующем ресканировании.
